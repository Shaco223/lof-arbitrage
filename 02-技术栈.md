# 02-技术栈

## 快速摘要

- 语言/框架：Python 3.11（拉取器） + uniCloud 云函数（Node.js） + uniapp（前端 Vue 3 / H5 → 小程序）
- 包管理器：pip（Python）/ npm（uniCloud / uniapp）
- 启动命令：见下文"常用命令"
- 测试命令：pytest（拉取器）/ uniCloud 控制台单元测试 / uniapp 本地预览
- 输出目录：`outputs/`

## 技术选型

### 总架构

```
本机 (Windows 11)
├── lof-fetcher/                     # Python 拉取器
│   ├── apscheduler 每分钟触发
│   ├── 拉 LOF 行情 / 重仓股行情 / 指数行情
│   ├── 算实时估算净值 + 真实溢价率
│   └── 通过 uniCloud HTTP 触发器写入数据库
│
uniCloud（免费阿里云空间）
├── 数据库（MongoDB 兼容）
│   ├── lof_meta             # 30 只 LOF 元数据（含业绩比较基准映射）
│   ├── lof_holdings         # 前十大重仓股权重
│   ├── lof_realtime         # 实时溢价（每分钟一条，仅保留当日）
│   ├── lof_history          # 历史溢价（每日收盘 1 条，保留 30 天）
│   └── alert_log            # 告警发送日志（防刷屏依赖）
├── 云函数（REST API）
│   ├── api/lof/list         # 列表 + 实时溢价
│   ├── api/lof/detail       # 详情 + 30 天分位
│   ├── api/lof/history      # 历史 K 线
│   └── ingest/realtime      # 拉取器写入接口（带签名）
└── 定时任务
    └── daily-calibrate      # 每日 18:00 用官方净值校准 + 算分位
│
uniapp（前端 Vue 3）
├── pages/index              # Dashboard：30 只 LOF 实时排行
├── pages/detail/[code]      # 详情页：曲线 + 分位 + 估算覆盖率
└── pages/settings           # 阈值 / 通道配置
```

### 各层技术选型

#### 本机拉取器（Python）

- 语言：Python 3.11
- 调度：apscheduler（盘中每分钟）
- HTTP：httpx（异步）
- 数据处理：pandas
- 配置：pydantic-settings + .env
- 日志：loguru
- 推送：Server酱（或邮件，二选一）

#### 后端 uniCloud

- 平台：uniCloud 阿里云免费空间
- 云函数运行时：Node.js 18
- 数据库：JQL（MongoDB 兼容）
- 鉴权：URL 化云函数 + 签名 token（拉取器 → uniCloud 走 ingest 接口需签名）
- 定时任务：uniCloud 内置 cron（每日 18:00 校准）

#### 前端 uniapp

- 框架：uniapp + Vue 3 + setup script
- UI：uni-ui
- 图表：ucharts（uniapp 生态推荐）
- 状态：pinia
- 编译目标：一期 H5；二期开启微信小程序构建

## 常用命令

```powershell
# 本机拉取器
cd lof-fetcher
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m fetcher.main           # 启动
pytest                           # 测试

# uniCloud（HBuilderX 内）
# 右键 cloudfunctions/ → 上传部署
# 右键 db_init.json → 初始化云数据库

# uniapp 前端
cd frontend
npm install
npm run dev:h5                   # 本地预览 H5
npm run build:h5                 # 打包 H5
npm run dev:mp-weixin            # 二期再开启
```

## 环境变量

| 变量名 | 用途 | 配置位置 |
| --- | --- | --- |
| UNICLOUD_INGEST_URL | uniCloud 拉取器写入接口 URL | lof-fetcher/.env |
| UNICLOUD_INGEST_TOKEN | uniCloud 写入签名 token | lof-fetcher/.env |
| SERVERCHAN_KEY | Server酱推送 key（可选） | lof-fetcher/.env |
| EMAIL_SMTP / EMAIL_USER / EMAIL_PASS | 邮件推送（可选，与 Server酱二选一） | lof-fetcher/.env |
| ALERT_PREMIUM_THRESHOLD | 溢价告警阈值，默认 0.05 | lof-fetcher/.env |
| ALERT_DISCOUNT_THRESHOLD | 折价告警阈值，默认 -0.02 | lof-fetcher/.env |
| ALERT_COOLDOWN_MINUTES | 同基金告警冷却分钟，默认 30 | lof-fetcher/.env |

## 关键路径

- 拉取器源码：`lof-fetcher/`
- uniCloud 工程：`uniCloud-aliyun/`（HBuilderX 默认目录）
- 前端源码：`frontend/`
- 配置：各模块根目录 `.env` / `manifest.json`
- 输出：`outputs/`（CSV / HTML 报表 / 截图）
- 资料：`assets/`（基金清单、业绩比较基准映射表草稿）

## 关键模块边界

- **拉取器（dev-004 后端）**：仅负责拉数据 + 算 IOPV + 写 uniCloud；不直接对前端开放接口。
- **uniCloud 云函数（dev-004 后端）**：仅负责数据存取 + REST API；不直接拉外部数据源。
- **uniapp 前端（dev-003 前端）**：仅消费 uniCloud REST API；不直接连 Python 拉取器。
- **测试（dev-005 测试）**：以"实时溢价精度 ≥ 90%"和"功能符合 PRD"为核心验收标准。
