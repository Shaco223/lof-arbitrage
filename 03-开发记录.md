### 2026-06-18 18:00｜流程裁决 #003：多角色工作区隔离

- 背景：dev-002/dev-003/dev-004/dev-005 原计划共用同一个物理目录开发，但 Git 工作区同一时间只能 checkout 一个分支，多个 Codex 会话会互相切换分支并污染工作状态。
- 裁决：采用 Git worktree 隔离。项目根目录仅由项目经理 dev-001 使用；各角色固定进入自己的 `.worktrees/<role>` 目录开发。
- 固定目录：
  - dev-002 产品经理：`.worktrees/dev-002-product`
  - dev-003 前端开发：`.worktrees/dev-003-frontend`
  - dev-004 后端开发：`.worktrees/dev-004-backend`
  - dev-005 测试：`.worktrees/dev-005-test`
- 执行结果：
  1. `.gitignore` 已加入 `.worktrees/`，防止本地 worktree 目录入库。
  2. 已创建四个角色 worktree，并固定绑定对应工作分支。
  3. dev-003 与 dev-005 的既有越位分支已在独立 worktree 中 rebase 到最新 `origin/main`。
  4. 旧 stash 已核验：dev-003/dev-005 的有效工作已在对应分支中；剩余 stash 仅作为迁移留证，禁止角色自行跨目录套用。
- 新纪律：
  - 任何角色禁止在项目根目录执行 `git checkout <自己的分支>`。
  - 任何角色只在自己的 worktree 中 `git fetch` / `git rebase origin/main` / `git push`。
  - dev-001 仍为唯一合并人，合并前从各角色远程分支审核差异并在根目录 squash merge。
  - 如需新增角色或新增长期分支，先由 dev-001 创建 worktree 后再派工。
